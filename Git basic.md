# local 정리
1. git init 아느이 모든 폴더는 git 관리대상
2. git의 3가지 영역은 working directory/ staging area/ repository
3. working directory 는 실제 작업(untracked, 수정시 modify)
4. staging area 는 커밋 특정버전 관리(staged)
5. Repository 커밋 확정(commited)
6. working directory 에서 working directory 갈 때 git add
7. staging area 에서 Repository 갈 때 git commit
8. git commit -m 은 메시지 같이 등록하는 옵션
9. 처음 git commit 할 때는 메일, 사용자 이름 등록

10. git status는 staging area 상태를 알려줌
11. git log는 repository 상태 알려줌
